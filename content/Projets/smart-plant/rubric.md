# 🌱 Projet Arduino – Plante Connectée

## 🧾 Pondération totale : **25 % du cours**

Ce projet est évalué en **3 étapes** progressives permettant de valider les compétences en électronique, programmation embarquée, développement web et intégration de systèmes.

---

# 🟢 Évaluation 1 — Arduino + Circuit + Communication (10%)

## 🎯 Objectifs
- Réaliser un circuit fonctionnel avec :
  - capteur de température/humidité (DHT11)
  - capteur d’humidité du sol
  - LEDs (indicateurs)
- Programmer l’Arduino pour :
  - lire les données
  - envoyer les données via Serial
  - recevoir des commandes simples (ex: activer LED)

## 📊 Critères (10%)

| Critère | Description | Points |
|--------|------------|--------|
| 🔌 Circuit | Montage correct (capteurs + LEDs bien branchés, résistances présentes) | 2 |
| 📥 Lecture des données | Lecture correcte des capteurs (valeurs cohérentes) | 3 |
| 🔄 Communication sortante | Données envoyées correctement via Serial (format clair) | 2 |
| 📡 Communication entrante | Réception et traitement de commandes (ex: allumer LED) | 2 |
| 🧠 Qualité du code | Code structuré, lisible, variables claires | 1 |

---

# 🔵 Évaluation 2 — Dashboard React (5%)

## 🎯 Objectifs
- Créer une interface utilisateur avec React ou Vanilla JS
- Afficher les données simulées ou reçues
- Offrir une interaction utilisateur (boutons, affichage dynamique)

## 📊 Critères (5%)

| Critère | Description | Points |
|--------|------------|--------|
| 🎨 Interface UI | Interface claire, lisible, organisée | 2 |
| 📊 Affichage des données | Température et humidité affichées correctement | 1 |
| 🎛️ Interaction | Boutons/contrôles fonctionnels (ex: activer pompe) | 1 |
| 🧠 Structure du code | Composants bien organisés, code propre | 1 |

---

# 🟣 Évaluation 3 — Intégration finale + Présentation (10%)

## 🎯 Objectifs
- Utiliser un backend fourni (API prête)
- Connecter :
  - Arduino → Backend
  - Backend → Frontend
- Présenter un système complet fonctionnel

## 📊 Critères (15%)

### 🔗 Intégration technique (5 pts)

| Critère | Description | Points |
|--------|------------|--------|
| 🔌 Arduino ↔ Backend | Envoi des données vers l’API fonctionnel | 2 |
| 📥 Backend ↔ React | Récupération et affichage des données | 2 |
| ⚙️ Fonctionnement global | Système cohérent, sans bugs majeurs | 1 |

---

### 🗣️ Présentation (5 pts)

| Critère | Description | Points |
|--------|------------|--------|
| 🧠 Compréhension | Capacité à expliquer le fonctionnement du système | 2 |
| 🧩 Architecture | Explication claire des composants (Arduino, API, React) | 1 |
| 🎯 Démonstration | Démo fonctionnelle du projet | 1 |
| 🗣️ Clarté | Communication claire et structurée | 1 |

---

# 📌 Remarques importantes

- Le projet doit être **fonctionnel à chaque étape**.
- Une attention particulière est portée à :
  - la **compréhension globale**
  - la **qualité du code**
  - la **logique d’intégration**
- Le travail en équipe doit être équilibré (chaque étudiant doit comprendre toutes les parties).

On évalue une interface pour éviter que ce soit l’utilisateur qui paye les erreurs.  

**Les utilisateurs :**

- ne lisent pas tout

- oublient

- cliquent vite

- font des erreurs

- sont parfois stressés, pressés, fatigués

> Le problème n’est jamais l’utilisateur
> c’est l’interface qui doit s’adapter à lui

**L’évaluation sert à :**

- détecter les points de confusion

- réduire la charge cognitive

- prévenir les erreurs

---

## Les grandes méthodes d’évaluation UI/UX  

### 1. Évaluation heuristique : Analyse experte basée sur des règles reconnues.  

[Télécharger la grille d’audit UI/UX](/420-411-Interfaces-Humain-Machine/files/Grille_Audit_UI_UX.xlsx)

### 2. Tests utilisateurs :

On observe de vrais utilisateurs accomplir des tâches :

- Trouver une info

- Acheter un produit

- Remplir un formulaire

**🎯 On mesure :**

- erreurs

- temps

- hésitations

- incompréhensions  

**5 utilisateurs ≈ 80 % des problèmes détectés**


**Outils**

- Maze (prototypes)

- Lookback

- Zoom / Meet + observation

- Papier + stylo

### 3. Tests A/B :

**Comparer deux versions :**

- bouton bleu vs vert

- texte court vs long

- formulaire long vs court

**Décision basée sur les données**

- Google Optimize (ou équivalent)

- Outils intégrés e-commerce
